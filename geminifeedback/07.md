看到这张图，我完全明白问题出在哪里了。

**异常形变的根本原因：**
1. **坐标系与矩阵冲突**：我们之前利用了 Three.js 的 `attach()` 方法来动态重组父子节点以实现旋转。但在动画期间，外层的 `<motion.group>`（由 Framer Motion 3D 控制）也在更新全局的世界矩阵。两者的矩阵运算发生碰撞，导致小块的 Scale 和 Position 发生严重畸变。
2. **React 渲染重置**：当点击或状态变更时，React 触发了重新渲染。因为我们之前在 JSX 中写了 `position={pos}`，React 会强制把变形过程中的小块**拉回初始坐标**，直接导致了模型撕裂。

**最完美的解决方案（纯数学驱动）：**
放弃“改变父子节点绑定”的物理做法，改为在 `useFrame` 中利用纯数学四元数 (`Quaternion`) 精确计算每一帧的局部旋转，并且避开 React 对坐标的强行干预。

下面是修复后的 `src/components/cube/MatrixCube.tsx`，彻底解决了形变问题，而且性能更好，还原更精准。

请用以下代码**完全替换** `src/components/cube/MatrixCube.tsx`：

```tsx
import React, { useRef, useMemo, useEffect } from 'react';
import { useFrame } from '@react-three/fiber';
import { RoundedBox } from '@react-three/drei';
import { motion } from 'framer-motion-3d';
import * as THREE from 'three';
import { useCubeStore } from '../../store/useCubeStore';
import { FACE_ROTATIONS, DEFAULT_VIEW_ANGLE, NORMAL_TO_FACE } from './constants';

const CUBIE_SIZE = 0.96;
const CUBIE_OFFSET = 1.0;

export const MatrixCube: React.FC = () => {
  const { selectedFaceId, setSelectedFaceId, twistEvent, clearTwistEvent } = useCubeStore();
  
  const mainGroupRef = useRef<THREE.Group>(null);
  
  // 存储所有 27 个小方块的 Mesh 引用
  const cubiesRef = useRef<THREE.Mesh[]>([]);
  
  // 记录纯数学动画的进度与状态
  const animState = useRef({
    isTwisting: false,
    axis: 'y' as 'x'|'y'|'z',
    targetAngle: 0,
    startTime: 0,
    duration: 600,
    activeCubies: [] as {
      mesh: THREE.Mesh;
      startPos: THREE.Vector3;
      startQuat: THREE.Quaternion;
    }[]
  });

  // 生成 27 个小方块初始坐标
  const initialPositions = useMemo(() => {
    const coords = [];
    for (let x = -1; x <= 1; x++) {
      for (let y = -1; y <= 1; y++) {
        for (let z = -1; z <= 1; z++) {
          coords.push(new THREE.Vector3(x * CUBIE_OFFSET, y * CUBIE_OFFSET, z * CUBIE_OFFSET));
        }
      }
    }
    return coords;
  }, []);

  const glassMaterial = useMemo(() => new THREE.MeshPhysicalMaterial({
    color: '#e0f2fe', metalness: 0.1, roughness: 0.1, transmission: 0.8, thickness: 1.5,
    clearcoat: 1.0, clearcoatRoughness: 0.1, transparent: true, opacity: 1,
  }), []);

  const coreMaterial = useMemo(() => new THREE.MeshStandardMaterial({
    color: '#38bdf8', emissive: '#0ea5e9', emissiveIntensity: 0.6,
  }), []);

  // 监听层扭动事件 (剥离了恶心的 attach 重组，改为纯粹的预计算)
  useEffect(() => {
    if (twistEvent && !animState.current.isTwisting) {
      const { axis, layer, angle, duration } = twistEvent;
      
      const activeCubies: any[] = [];
      
      cubiesRef.current.forEach(mesh => {
        if (!mesh) return;
        // 查找当前处于旋转层的小方块
        const posValue = mesh.position[axis];
        if (Math.abs(posValue - (layer * CUBIE_OFFSET)) < 0.5) {
          activeCubies.push({
            mesh,
            startPos: mesh.position.clone(),
            startQuat: mesh.quaternion.clone()
          });
        }
      });

      // 初始化动画参数
      animState.current = {
        isTwisting: true,
        axis,
        targetAngle: angle,
        startTime: performance.now(),
        duration: duration || 600,
        activeCubies
      };
    }
  }, [twistEvent]);

  // 每帧计算矩阵，保证绝对顺滑与精准
  useFrame((state) => {
    if (animState.current.isTwisting) {
      const { axis, targetAngle, startTime, duration, activeCubies } = animState.current;
      const now = performance.now();
      
      // 计算进度 (0 到 1)
      let progress = (now - startTime) / duration;
      let finished = false;
      if (progress >= 1.0) {
        progress = 1.0;
        finished = true;
      }

      // 平滑缓动 (easeInOutQuad)
      const ease = progress < 0.5 ? 2 * progress * progress : 1 - Math.pow(-2 * progress + 2, 2) / 2;
      const currentAngle = ease * targetAngle;

      // 生成该帧的旋转四元数
      const axisVec = new THREE.Vector3(
        axis === 'x' ? 1 : 0,
        axis === 'y' ? 1 : 0,
        axis === 'z' ? 1 : 0
      );
      const pivotQuat = new THREE.Quaternion().setFromAxisAngle(axisVec, currentAngle);

      activeCubies.forEach(({ mesh, startPos, startQuat }) => {
        // 1. 坐标沿原点公转
        const newPos = startPos.clone().applyQuaternion(pivotQuat);
        mesh.position.copy(newPos);
        
        // 2. 本身姿态自转叠加
        const newQuat = pivotQuat.clone().multiply(startQuat);
        mesh.quaternion.copy(newQuat);

        // 3. 动画结束时，强制修正浮点数误差，确保精准对齐不会散架
        if (finished) {
          mesh.position.x = Math.round(mesh.position.x / CUBIE_OFFSET) * CUBIE_OFFSET;
          mesh.position.y = Math.round(mesh.position.y / CUBIE_OFFSET) * CUBIE_OFFSET;
          mesh.position.z = Math.round(mesh.position.z / CUBIE_OFFSET) * CUBIE_OFFSET;
          mesh.quaternion.normalize();
        }
      });

      // 动画结束，回收事件
      if (finished) {
        animState.current.isTwisting = false;
        clearTwistEvent();
      }
    }

    // 待机呼吸微动 (避开被旋转事件污染)
    if (!selectedFaceId && mainGroupRef.current && !twistEvent && !animState.current.isTwisting) {
      mainGroupRef.current.position.y = Math.sin(state.clock.elapsedTime * 1.5) * 0.05;
    } else if (mainGroupRef.current) {
      mainGroupRef.current.position.y = THREE.MathUtils.lerp(mainGroupRef.current.position.y, 0, 0.1);
    }
  });

  const handleCubieClick = (e: any) => {
    e.stopPropagation();
    if (twistEvent || animState.current.isTwisting) return; 
    if (e.face) {
      const normal = e.face.normal.clone().round();
      const faceId = NORMAL_TO_FACE[`${normal.x},${normal.y},${normal.z}`];
      if (faceId) setSelectedFaceId(faceId);
    }
  };

  const targetRotation = selectedFaceId ? FACE_ROTATIONS[selectedFaceId] : DEFAULT_VIEW_ANGLE;

  return (
    <motion.group
      ref={mainGroupRef}
      animate={{ rotateX: targetRotation[0], rotateY: targetRotation[1], rotateZ: targetRotation[2] }}
      transition={{ type: 'spring', stiffness: 60, damping: 15 }}
    >
      {initialPositions.map((pos, idx) => {
        const isCore = pos.x === 0 && pos.y === 0 && pos.z === 0;
        return (
          <RoundedBox
            key={idx}
            // 【核心修复】：脱离 React JSX 的坐标强绑定，改为挂载时仅初始化一次
            ref={(el: any) => {
              if (el && !cubiesRef.current[idx]) {
                el.position.copy(pos);
                cubiesRef.current[idx] = el;
              }
            }}
            args={[CUBIE_SIZE, CUBIE_SIZE, CUBIE_SIZE]}
            radius={0.05}
            smoothness={4}
            onClick={handleCubieClick}
            material={isCore ? coreMaterial : glassMaterial}
          />
        );
      })}
    </motion.group>
  );
};
```

### 做了哪些根本性修复？
1. **脱离了 React DOM 的约束 (`el.position.copy(pos)`)**: 之前 `position={pos}` 导致只要魔方组件重新渲染（比如你点击了面板导致状态改变），小方块就会被 React 强行弹回未扭动的原始位置，导致“撕裂形变”。现在只在初次挂载时赋予坐标，后续全由 Three.js 接管。
2. **废弃 `group.attach` 切换**：之前为了让层旋转，频繁把 Mesh 在不同的父级 Group 中拔插，严重干扰了 R3F 的矩阵更新。
3. **引入了纯数学四元数旋转 (`Quaternion`)**：现在直接算出旋转多少度，然后在原位置自转+公转，每一帧极为顺滑，并在结束时使用 `Math.round` 强制纠正浮点数偏离。
