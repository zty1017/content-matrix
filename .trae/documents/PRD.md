# 产品需求文档 (PRD) - Content Matrix 前端 Web Demo

## 1. 目标与背景
本项目旨在为 Content Matrix 创建一个 3-5 分钟可演示的 Web Demo。
核心叙事：用户刷到一条重庆美食视频，使用 Content Matrix 转换为结构化内容资产。随后用户刷到相关重庆烤肉/旅行视频，系统能将之前保存的资产召回，并影响新视频的决策判断卡。

## 2. 核心功能需求
- **视图 1：主演示页**
  - 左侧：重庆美食视频卡（展示视频信息及“AI 收藏 / 变成资产”按钮）。
  - 中栏：资产回流与用户上下文切换（展示资产化流程，通过 300-600ms 的 UI 延迟模拟生成过程，以及 3 个用户上下文：低预算、效率优先、本地文化）。
  - 右侧：决策判断卡（展示基于上下文和历史资产影响生成的决策卡内容，包括影响类型高亮）。
- **视图 2：资产库简版**
  - 展示已保存的重庆美食资产、决策卡快照及后续视频的引用。
- **视图 3：调试 JSON 面板**
  - 展示当前 `source resolution`、`VideoContentAsset`、`ReconstructionTask` 和 `SavedReconstructionSnapshot` 数据。

## 3. 非功能需求与约束
- **本地优先执行方案**：不调用真实抖音 API，不读取真实收藏夹，不实时调用 LLM。
- **数据源**：使用后端 fixture/runtime 中的 mock 流程和特定 ID 数据。
- **界面与交互**：
  - 风格：背景浅灰蓝或浅渐变，主色蓝紫渐变，卡片白色圆角轻阴影。
  - 影响类型颜色：`preference_adaptation` (绿色/青色)、`conflict_warning` (橙色)、`supplement` (蓝色)。
  - UI 体验：一眼能看出历史资产影响了新判断。
