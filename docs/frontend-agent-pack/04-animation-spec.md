# 04. 动效规范

## 1. 总原则

需要：魔方真实层旋转、内容块注入、资产陈列柜唤醒、保存入柜仪式感、工坊拆解重组。

避免：粒子雨、大面积霓虹、背景太暗、闪烁过强、动画时间过长、文字区域晃动。

## 2. 时间规范

```text
一键体验完整流程：8～12 秒
普通任务重构流程：3～6 秒
局部补充更新：0.8～1.5 秒
保存入柜：1～2 秒
资产预览取出 / 放回：0.6～1 秒
工坊多魔方拆解：3～5 秒
工坊新魔方生成：4～7 秒
顶部模式切换：300～600ms
陈列柜展开 / 收起：300～500ms
```

所有超过 2 秒的流程，应提供“立即查看结果”。

## 3. 主魔方待机

结果态 idle：

- 极小幅度上下悬浮；
- 低频冷蓝白呼吸光；
- 偶尔有 1～2 个小格轻微闪动；
- 不持续大幅旋转；
- 不影响点击和阅读。

## 4. 一键体验流程动画

```text
选择示例魔方
→ 来源识别
→ 内容拆解
→ 历史资产陈列柜自动亮起
→ 相关资产小魔方浮出
→ 抽取内容块
→ 内容块注入主魔方核心
→ 内部光路分配到各语义面
→ 主魔方层旋转、打乱、还原
→ 结果停在 3/4 视角
```

## 5. 魔方层旋转

必须看起来像标准三阶魔方层旋转。  
不做用户手动拧层，不要求完整魔方游戏状态机，但视觉必须明确是 3×3×3 层在转。

推荐序列：

```text
解析中：
- U 层 90°
- R 层 -90°
- F 层 90°

资产增强后：
- M 层 90°
- U 层 -90°
- R 层 90°

最终还原：
- R 层 -90°
- U 层 90°
- F 层 -90°
- 整体缓动到 3/4 视角
```

## 6. 历史资产注入动画

采用：先依次点亮相关资产，再分批并行注入内容块。

节奏：

```text
1. 当前相关层亮起
2. 资产 A 浮出：提供预算偏好
3. 资产 B 浮出：提供排队经验
4. 资产 C 浮出：提供证据补充
5. 第一批内容块注入：偏好 / 约束
6. 第二批内容块注入：证据 / 原始线索
7. 第三批内容块注入：推理 / 标签
8. 主魔方层旋转、打乱、还原
```

内容块先进入主魔方核心，再沿内部光路分配到各语义面。

## 7. 叙事型状态栏

采用阶段胶囊节点：

```text
[来源识别 ✓] [内容拆解 ✓] [资产增强 ●] [魔方成型]
```

状态：

- 未开始：浅灰玻璃；
- 进行中：柔和发光、呼吸；
- 已完成：小勾 + 低亮高光；
- 待确认：琥珀细边；
- 失败：低饱和红细边。

## 8. 保存入柜动画

```text
主魔方复制出一个小型玻璃副本
→ 左侧陈列柜半展开
→ 小魔方沿弧线飞入“最近生成”层
→ 底座亮起
→ 铭牌生成
→ 中央主魔方保留结果态
```

## 9. 资产预览取出 / 放回

点击陈列柜资产：

```text
资产魔方从展位浮出
→ 移动到中央预览位
→ 当前任务魔方变成半透明背景 + 任务锚点
→ 右侧面板显示资产详情
```

返回当前任务：

```text
预览魔方缩小
→ 沿原路径回展位
→ 展位底座短亮
→ 当前任务魔方恢复前景
```

## 10. 人工补充局部更新

```text
对应面轻微发光
→ 新增内容块以玻璃碎片形式吸附到该面
→ 相邻相关面轻微联动
→ 右侧提示：已局部更新
```

## 11. 高风险 / 草稿动画

高风险阻塞：

- 魔方局部停止还原；
- 呈现“接近完成但未闭合”的状态；
- 对应面低饱和红细边；
- 右侧显示确认入口。

草稿转正：

```text
半透明未闭合
→ 边缘闭合
→ 内容块吸附稳定
→ 琥珀弱光退去
→ 冷蓝白高光出现
→ 移动到正式资产层
```

## 12. 重播动画

结果态提供“重播重构过程”。只重放前端动画，不重新请求后端生成。必须重播历史资产浮出并注入主魔方过程。

## 13. 跳过动画

点击后：

- 魔方动画快速收束到最终形态；
- 阶段胶囊全部完成；
- 右侧面板显示结果；
- 保留轻量提示：已跳过动画。
