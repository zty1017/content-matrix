# 06. MVP 范围与 Roadmap

## 1. MVP 核心目标

用本地 fallback 数据 + 后端 API 双轨，稳定展示完整内容魔方体验。  
不追求现场真实抖音解析，优先确保前后端跑通和比赛演示稳定。

## 2. MVP 必须实现

- 中央标准 3×3×3 魔方；
- 半透明琉璃材质；
- 整体拖拽旋转；
- 点击面打开右侧详情；
- 系统控制层旋转，打乱 / 还原真实可见；
- source 面显示封面 / 关键帧；
- 三个首页示例魔方；
- 一键体验自动跑完；
- 历史资产陈列柜亮起、浮出、注入；
- 保存入柜动画；
- 左侧上下分层玻璃陈列柜；
- 右侧过程 / 详情面板；
- 轻量魔方工坊；
- 后端 API + fallback；
- localStorage 轻量持久化；
- Demo 控制面板。

## 3. MVP 可以模拟

- 魔方工坊真实重组结果；
- 内容块权重对生成的真实影响；
- 多轮版本对比；
- 复杂关系图；
- 片段级视频跳转；
- 完整 ASR 时间轴；
- 工坊草稿持久化；
- 高级导出。

## 4. 明确不做

### 完整视频播放器

首版只做封面 / 关键帧 / 小预览位 + 结构化解析信息。

Roadmap：

- 本地视频路径播放；
- ASR 时间戳联动；
- 证据跳转原始片段；
- 关键帧时间轴；
- 视频片段裁剪 / 引用。

### 登录 / 用户系统

首版不做登录，只做演示上下文。

Roadmap：

- 登录 / 注册；
- 用户资产隔离；
- 多端同步；
- 真实抖音账号授权；
- 团队协作空间；
- 权限管理。

### 完整设置页

首版只做 Demo 控制面板。

Roadmap：

- Provider 配置；
- 真实 ASR / LLM 接入开关；
- 动画偏好；
- 资产库展示密度；
- 隐私与数据清理；
- 导出格式偏好。

## 5. 魔方工坊 Roadmap

- 内容块权重系统：拖近 / 拖远影响权重，强引用 / 弱引用 / 排除。
- 多版本重组：每次重新生成保留版本，支持对比和回滚。
- 工坊草稿：未完成组合可保存，下次继续编辑。
- 真实后端重组接口：多资产 ID + 目标 + 模板 + 内容块选择。
- 细粒度内容块编辑：摘要、证据、标签、脚本钩子、归类。
- 关系图增强：一阶 / 二阶 / 全局关系图。
- 自动推荐重组方案：推荐知识卡 / 决策卡 / 脚本卡。
- 工坊结果可继续拆解：递归式资产演化。
- 协作与批注：队友评论、证据可靠性标记、多人确认。
- 导出能力：Markdown、PDF、飞书文档、分享卡片、展示海报。

## 6. 资产陈列柜 Roadmap

- 虚拟滚动；
- 分页；
- 资产批量编辑；
- 批量标签；
- 智能去重；
- 资产质量评分；
- 资产生命周期：草稿 / 正式 / 归档；
- 资产空间搜索；
- 更强关系网络。

## 7. 主魔方 Roadmap

- 更精确魔方状态机；
- 用户可手动拧层的探索模式；
- 每个小格真实绑定数据块；
- 更丰富材质和环境；
- 多魔方同屏协作；
- 片段级证据在魔方上定位。

## 8. API Roadmap

- 真实抖音解析；
- 真实 ASR；
- OCR / 关键帧；
- 真实 LLM；
- 异步任务；
- WebSocket / SSE 进度；
- 数据库；
- 用户认证；
- 多用户资产隔离。

## 9. 演示优先级

高光优先：

1. 首页三个示例魔方；
2. 一键体验；
3. 历史资产陈列柜亮起；
4. 相关资产浮出并注入主魔方；
5. 魔方层旋转打乱还原；
6. 六面结果；
7. 保存入柜；
8. 工坊轻量重组。

如果时间不足，优先保证 1～7，工坊做轻量可演示。
