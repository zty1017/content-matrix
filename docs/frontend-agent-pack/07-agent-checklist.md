# 07. 本地 AI Agent 执行检查清单

## 1. 不要误解

- 不要做普通上传页。
- 不要做普通卡片列表资产库。
- 不要把魔方做成非标准形态；必须是 3×3×3。
- 不要做 3×3×4 或异形魔方。
- 不要把所有信息塞到魔方面上。
- 不要让用户手动拧层。
- 不要依赖真实抖音链接在线解析作为演示主路径。
- 不要强依赖后端成功；必须有 fallback。
- 不要做深色霓虹赛博风。
- 不要把陈列柜做成普通列表或表格。

## 2. 必须先做

- React + TypeScript 项目初始化。
- 类型定义。
- 本地 demo 数据。
- AppShell 布局。
- 中央魔方占位。
- 左侧陈列柜占位。
- 右侧面板占位。
- 顶部导航。
- 底部输入条。
- 阶段胶囊状态栏。

## 3. 魔方检查

必须满足：

- 标准三阶魔方；
- 3×3×3 结构清晰；
- 整体可拖拽旋转；
- 点击面可选中；
- source 面支持封面；
- 系统动画有层旋转；
- 打乱还原看起来真实；
- 结果停在 3/4 视角。

不要求：

- 用户手动拧层；
- 完整魔方复原算法；
- 每个小格真实绑定数据。

## 4. 陈列柜检查

必须满足：

- 左侧；
- 收起 / 半展开 / 全展开；
- 上下分层；
- 玻璃陈列柜风格；
- 小魔方有底座和铭牌；
- 搜索和筛选；
- 保存入柜动画；
- 资产预览用中央区域；
- 多选和拖拽进入工坊。

## 5. 右侧面板检查

必须满足：

- 重构时显示过程面板；
- 结果态显示面详情；
- 支持展开 / 半收起 / 收起；
- 六个面都有对应内容；
- 支持轻量复制；
- 操作按钮分全局和面级。

## 6. 一键体验检查

必须满足：

- 三个示例魔方；
- 默认探店决策；
- 一键体验自动跑完；
- 有跳过动画；
- 有重播动画；
- 资产陈列柜相关资产参与注入；
- 魔方打乱还原；
- 最终六面可查看。

## 7. 后端接入检查

必须接入或预留：

```text
GET /health
POST /api/v1/source/resolve
POST /api/v1/tasks
GET /api/v1/cube/tasks/{task_id}
GET /api/v1/cube/tasks/{task_id}/progress
POST /api/v1/tasks/{task_id}/generate-card
POST /api/v1/tasks/{task_id}/save-snapshot
GET /api/v1/snapshots
POST /api/v1/local-video/parse
```

必须有 API base URL 配置、timeout、错误处理、fallback、Demo 控制面板测试连接。

## 8. fallback 检查

必须内置：

- 探店决策 fallback；
- 职场短剧 fallback；
- 财经知识 fallback；
- 历史资产 fallback；
- 工坊 fallback；
- localStorage 保存 fallback 资产。

## 9. 工坊检查

首版必须：

- 独立工坊视图；
- 多资产进入；
- 魔方拆解；
- 内容块语义分区；
- 新魔方胚胎；
- 一句话目标；
- 快捷模板；
- 输出风格；
- 新魔方生成；
- 结果态；
- 保存入柜。

可以模拟：权重真实影响、多版本、工坊草稿、后端重组。

## 10. UI 风格检查

必须：浅色冷调、玻璃琉璃、冷蓝白高光、克制科技感、留白充足、动效轻盈。

避免：大面积黑底、霓虹过载、粒子雨、文字太多、拥挤布局、过长动画。

## 11. 演示前检查

- 启动前端。
- 启动后端。
- Demo 控制面板测试连接。
- 强制 fallback 也能跑。
- 一键体验能完整播放。
- 跳过动画可用。
- 重播动画可用。
- 保存入柜可用。
- 刷新后 localStorage 资产仍在。
- 工坊轻量流程可演示。
- 1366×768 不崩。
- 1920×1080 展示最佳。
