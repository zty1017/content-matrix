# 01. Content Matrix 前端 UI/UX 规格

## 1. 产品定位

Content Matrix 前端是一个以 **内容魔方** 为核心的空间化内容重构工作台。

它不是传统的：

```text
上传页 → loading → 结果页
```

而是：

```text
内容进入魔方 → 魔方解析 / 打乱 / 吸收历史资产 / 还原 → 六面结果可交互 → 保存为新资产魔方 → 后续可拆解重组
```

核心用户感知：

- 内容不是被简单总结，而是被拆解、重构、资产化。
- 历史资产不是普通列表，而是会被系统自动调用增强当前任务。
- 每个结果都是一个可以回看、复用、拆解、重组的内容魔方。

## 2. 总体视觉风格

已确认风格：浅色冷调、玻璃 / 琉璃质感、克制科技感、空间化工作台、轻量数据流。避免赛博霓虹、粒子雨、深色压迫。

主魔方材质：

- 半透明冷白 / 冰蓝琉璃；
- 边缘有细微折射高光；
- 每个小方块之间保留清晰缝隙；
- 内部有弱发光骨架，强调 3×3×3 结构；
- `source` 面可以嵌入视频封面；
- 其他面使用低密度图标 / 标签 / 状态点。

## 3. 顶层页面模式

顶部导航采用三个主入口：

```text
当前魔方｜资产陈列柜｜魔方工坊
```

- 当前魔方：新输入、新解析、当前内容重构。
- 资产陈列柜：回顾、搜索、预览、复用历史资产。
- 魔方工坊：多资产拆解、重组、生成新资产。

切换时使用 300～600ms 的短镜头空间过渡，不做长动画。

## 4. 首页初始态

首页不是空白上传页，而是示例魔方展台：

```text
中央：一个大示例魔方
两侧：两个半透明候选示例魔方
下方：玻璃输入条
右侧：当前示例说明 / 一键体验
左侧：收起的资产陈列柜边栏
```

三个示例：

1. 探店决策魔方：默认高亮，适合主线演示。
2. 职场短剧拆解魔方：展示剧情结构、广告植入、脚本钩子。
3. 财经知识解析魔方：展示观点摘要、证据链、风险提示。

示例魔方视觉差异采用：弱颜色区分 + 正面内容封面 + 极简图标纹理，整体保持冷调玻璃琉璃统一。

一键体验默认自动完整跑完，提供弱化的“立即查看结果”按钮。结束后主魔方停在三面可见的 3/4 视角，右侧默认展示 `primary_card`。

## 5. 主魔方交互

用户可控：

- 整体拖拽旋转；
- 点击某个面查看详情；
- 点击某个面的 `+` 进行补充 / 修正；
- 拖拽资产魔方参与复用 / 重组；
- 点击空白区域回到默认 3/4 视角。

系统可控：

- 层旋转；
- 打乱；
- 还原；
- 内容块注入；
- 多魔方拆解重组；
- 保存入柜副本飞行动效。

不开放：首版不允许用户手动拧动某一层。层旋转只由系统动画控制。

拖拽规则：

- 拖拽中自由旋转；
- 松手后有轻微惯性；
- 接近标准视角时弱吸附；
- 点击某个面时自动旋转到该面正视角；
- 点击空白区域时回到 3/4 视角；
- 右侧详情面板保持最近选中的面，不因点击空白而切回概览。

面点击规则：

- 整个面作为点击区域；
- 3×3 小格只做视觉、状态、内容纹理表达；
- 小格不作为首版独立业务点击区域。

## 6. 六个魔方面

| face_id | UI 名称 | 作用 |
|---|---|---|
| source | 原始内容 | 视频封面、来源、解析状态 |
| primary_card | 主重构卡片 | 最终结果 / 主卡片 |
| related_assets | 历史资产关联 | 参与增强的资产 |
| inferences | 推理与待确认 | AI 推理、风险、假设 |
| evidence | 证据链 | ASR、OCR、标题、历史资产证据 |
| snapshot | 保存入柜 | 保存为内容资产 |

魔方面信息密度：低密度展示，只放标题、图标、1～2 个关键词、状态。例外：`source` 面可放视频封面 / 关键帧。

3×3 小格 MVP 不强制绑定真实数据块，只做抽象状态纹理。

## 7. 输入入口

采用组合模式：

```text
下方悬浮输入条 + 魔方拖拽投放区 + 每个面局部补充入口
```

下方输入条：

```text
[粘贴抖音链接 / 输入你想重构成什么 / 拖入视频] [开始重构]
```

演示不要求现场真实使用抖音链接解析，优先确保后端 fixture / fallback 跑通。

## 8. 右侧详情 / 过程面板

右侧面板有三种状态：

- 展开：完整详情；
- 半收起：标题、状态、关键摘要、主操作；
- 收起：只保留竖向玻璃标签。

重构过程中，右侧面板保留，并变为实时过程面板。默认卡片式步骤流；点击展开可看日志或节点关系。

步骤：来源识别、内容拆解、历史资产增强、魔方重构、待确认事项。

## 9. 六面右侧详情结构

### primary_card

```text
主重构卡片
├─ 标题
├─ 类型：决策建议 / 知识解释 / 脚本复用 / 经验卡
├─ 一句话结论
├─ 关键理由
├─ 行动建议
├─ 证据引用
└─ 操作：[保存为资产] [补充目标] [查看证据] [重新生成] [复制卡片]
```

### source

```text
原始内容
├─ 视频封面 / 小型预览
├─ 标题
├─ 来源类型：抖音链接 / 本地视频 / 示例素材 / 手动输入
├─ 时长 / 发布时间 / 作者，如果有
├─ 解析状态
├─ 自动抽取结果：标题 / ASR 摘要 / 关键片段 / 内容类型 / 广告植入或风险提示
└─ 操作：[补充上下文] [查看完整 ASR] [查看证据链] [重新解析] [复制 ASR 摘要]
```

首版不做完整播放器，只做封面 / 关键帧 / 小预览位。

### related_assets

```text
历史资产关联
├─ 关系概览：已参考 N 个历史资产、贡献类型、关联强度
├─ 参与增强的资产：标题、类型、贡献内容、注入位置、是否人工确认
├─ 注入内容块：偏好块、证据块、约束块、标签块
└─ 操作：[查看关系] [加入更多资产] [进入魔方工坊] [禁用某个引用]
```

### evidence

```text
证据链
├─ 证据概览
├─ 原始视频证据
├─ 历史资产证据
├─ 用户补充证据
├─ 待确认证据
└─ 操作：[补充证据] [标记已确认] [查看原始片段] [复制证据链摘要]
```

### inferences

```text
推理与待确认
├─ 推理概览
├─ AI 推理列表：内容、类型、依据来源、置信度、状态
├─ 待确认事项
├─ 风险提示
└─ 操作：[确认推理] [否定推理] [补充上下文]
```

### snapshot

```text
保存为内容资产
├─ 保存预览
├─ 资产铭牌：标题、说明、类型、状态
├─ 分类标签：自动标签、用户标签、场景分类
├─ 保存影响说明
└─ 操作：[保存入柜] [保存为草稿] [编辑铭牌] [取消] [复制资产摘要]
```

## 10. 人工补充

每个面有两处入口：魔方面小型 `+`、右侧详情面板“补充 / 修正”。

补充完成：

- 默认触发局部轻量重构；
- 不重新跑完整打乱还原；
- 用户主动点“重新生成完整结果”时才完整重构。

标记规则：

```text
AI 自动抽取：细蓝色光点
人工补充：浅紫 / 银白小印章
人工确认：小勾印记
待确认：琥珀色细边
高风险：低饱和红色细边
```

## 11. 风险与草稿

普通待确认：可继续生成，结果保留待确认标记。  
高风险：不直接生成最终魔方，停在 `needs_confirmation / blocked` 状态；用户确认后继续，或保存为草稿魔方。

草稿魔方：放在陈列柜“待确认”层，半透明，底座琥珀色弱光，默认不参与自动增强，除非用户手动选择。
